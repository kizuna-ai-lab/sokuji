# P6: macOS MLX TTS Lane Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** On Apple Silicon macOS, the `moss-tts-nano` and `qwen3-tts-{0.6b,1.7b}` cards run through `mlx-audio` (Metal) instead of the ONNX CPU path (spec D5). The MLX lane is a NEW backend + extra catalog deployment rows on the SAME cards — no new model ids, no protocol change, zero behavior change on Linux/Windows.

**Architecture:** A new backend module `mlx_tts.py` registers `MlxAudioTtsBackend` (NAME `mlx_audio_tts`) into the shared `backends` registry. It lazy-imports `mlx_audio` inside `load()` (so the Linux dev venv stays importable — a missing wheel raises `BackendLoadError`, which the resolver treats as a normal fallback to the ONNX cpu row) and adapts `mlx_audio`'s `model.generate()` chunk generator to the existing TTS streaming contract (`generate` / `generate_stream` / `set_voice` / `set_builtin_voice` / `set_speaker` / `set_language` / `sample_rate`) via a one-function seam `_extract_samples(chunk)`. `tts_engine.py` is UNTOUCHED — the adapter lives entirely in the backend. Each MLX-lane card gains ONE `Deployment("mlx_audio_tts", "gpu-metal", <same compute_type as the card's ONNX rows>, <mlx-community repo>, ..., platforms=("macos",), requires_apple_silicon=True)`, ranked above `cpu` by tier so mac resolves MLX first; the ONNX cpu rows stay as the mac fallback. P3's `_platform_ok` (applied inside `resolve_tts`) and the `mlx_audio_tts`-not-installed gate both drop the MLX row on non-mac. `accel._installed()` gains one dependency-map entry; `_tier_available("gpu-metal")` already lights on Apple Silicon (reused, not changed). `native_models.download_specs` becomes platform-aware for these three cards: Apple Silicon macOS fetches the single self-contained MLX repo, every other platform keeps the ONNX repos. The MLX rows REUSE the card's existing `compute_type` label ("fp32") so `seen_cts` stays length-1 and `_h_models_catalog` emits no new TTS `variantIds` — keeping the wire protocol identical to today's single-variant TTS behavior on every OS.

**Tech Stack:** `mlx-audio==0.4.4` (Apple-Silicon-only pip wheel; `from mlx_audio.tts.utils import load_model`, `model.generate(text=, voice=, speed=)` is a chunk generator), stdlib `platform` (via P3's `accel.current_platform()`), `numpy`, `soundfile` (already pinned; used to stage clone reference clips). No new Linux dependency — the pin is added with an environment marker so it installs only on `darwin`/`arm64`.

## Global Constraints

- **Depends on P3** (`2026-07-06-p3-catalog-platforms.md`): `catalog.Deployment.platforms` / `requires_apple_silicon` fields, `accel.current_platform()`, `accel._platform_ok`, and the platform filtering already wired into `resolve_tts` and `_h_models_catalog`'s tiers loop. This plan builds on all of them; do NOT re-implement the platform predicate.
- **Zero behavior change on Linux/Windows**: on any non-Apple-Silicon machine the MLX row is filtered out (by `_platform_ok` AND by the `mlx_audio_tts`-not-installed gate), the MLX repo is never downloaded, and no new protocol field is emitted. Every test monkeypatches `accel.current_platform` (never relies on the host OS) so the suite passes identically on the Linux dev box.
- **MLX rows REUSE the card's compute_type** ("fp32"). A distinct label would make `seen_cts` length-2 and emit TTS `variantIds` for the first time — untested renderer territory. The repo/quant reality lives in the artifact; the badge label is cosmetic (a follow-up, see checklist).
- **Sidecar runtime stays torch-free** (`tests/test_torch_free_gate.py` must keep passing): `mlx_tts.py` imports only `os`/`tempfile`/`time`/`numpy`/`.backends` at module level; `mlx_audio` and `soundfile` are imported lazily inside methods.
- **No interface change to `tts_engine.py`**: the chunk-generator→streaming adaptation is done inside the backend.
- All comments/docs in English. Conventional commit messages. No `git push` / no HF uploads (operator-gated; none are needed here — the MLX repos are upstream `mlx-community/*`).
- `mlx-audio` cannot be installed or exercised on the Linux dev machine; every MLX interaction is faked via a `sys.modules` stub (mirrors `test_ct2_opus`'s pattern). Real-hardware behavior is deferred to the checklist at the end.

```python
"""macOS MLX TTS backend (spec D5): Qwen3-TTS and MOSS-TTS-Nano on Apple
Silicon via the mlx-audio package. Registered under the NAME "mlx_audio_tts".

mlx-audio's Model.generate() is a generator that yields audio chunks; this
backend adapts those chunks to the shared TTS streaming contract (generate /
generate_stream + set_voice / set_builtin_voice / set_speaker / set_language /
sample_rate), so tts_engine drives it exactly like the ONNX streaming backends.

mlx-audio is an Apple-Silicon-only (Metal) wheel. It is imported LAZILY inside
load(), so importing this module on the Linux/Windows dev venv never touches it;
load() raises BackendLoadError when the wheel is absent, which the accel
resolver treats as a normal fallback to the ONNX cpu row."""
import os
import tempfile
import time

import numpy as np

from .backends import register_backend, BackendLoadError

# Short id -> mlx-community HF repo. Unknown ids (the usual case: the catalog
# Deployment.artifact already IS the mlx-community repo) pass straight through.
MLX_TTS_REPOS: dict[str, str] = {}


def _extract_samples(chunk) -> np.ndarray:
    """Adapt one mlx-audio generation chunk to a 1-D float32 numpy array.

    mlx-audio's generate() yields GenerationResult-like objects; across releases
    the audio payload has appeared under `.audio` (an mlx array), with `.samples`
    / `.wav` as aliases, and the object is sometimes the raw array itself. This
    single function is the seam to fix if the real attribute name — or the
    mlx->numpy conversion (a live mlx array may need mx.eval()/np.array()) —
    differs on Apple hardware. See the deferred mac checklist."""
    data = chunk
    for attr in ("audio", "samples", "wav"):
        val = getattr(chunk, attr, None)
        if val is not None:
            data = val
            break
    return np.asarray(data, dtype=np.float32).reshape(-1)


@register_backend
class MlxAudioTtsBackend:
    NAME = "mlx_audio_tts"
    STREAMING = True
    CLONES = True

    def __init__(self):
        self._model = None
        self._voice = None        # builtin voice name OR a staged reference-clip path
        self._ref_path = None     # temp wav backing a clip clone (cleaned on unload)
        self._ref_text = ""       # reference transcript (ICL) — stored; see checklist
        self._lang = ""
        self.sample_rate = 24000

    # ---- loading -----------------------------------------------------------
    def load(self, model_ref: str, device: str, compute_type: str) -> None:
        self._model = None
        try:
            from mlx_audio.tts.utils import load_model
            repo = MLX_TTS_REPOS.get(model_ref, model_ref)
            # The repo is pre-downloaded by native_models; load_model resolves it
            # from the HF cache. device/compute_type are ignored — mlx runs Metal.
            self._model = load_model(repo)
            self.sample_rate = int(getattr(self._model, "sample_rate", 24000) or 24000)
        except Exception as e:  # missing wheel / not Apple Silicon / bad repo → resolver falls back
            self._model = None
            raise BackendLoadError(str(e))

    # ---- voice / language --------------------------------------------------
    def set_language(self, lang) -> None:
        self._lang = lang or ""

    def set_speaker(self, sid) -> None:
        pass  # mlx voices are named, not a numeric speaker range

    def set_builtin_voice(self, name: str) -> None:
        self._clear_ref_file()
        self._voice = str(name)
        self._ref_text = ""

    def set_voice(self, audio, sr, ref_text: str = "") -> None:
        """Zero-shot clone: mlx-audio takes a reference-clip PATH as `voice`, so
        stage the clip to a temp float32 wav and point `voice` at it. The
        reference transcript is stored for ICL models but NOT yet threaded into
        generate() (mlx-audio's documented signature is text/voice/speed) — see
        the deferred mac checklist."""
        import soundfile
        wav = np.asarray(audio, dtype=np.float32)
        if wav.ndim > 1:
            wav = wav.mean(axis=1).astype(np.float32)
        self._clear_ref_file()
        fd, path = tempfile.mkstemp(prefix="sokuji_mlx_ref_", suffix=".wav")
        os.close(fd)
        soundfile.write(path, wav, int(sr))
        self._ref_path = path
        self._voice = path
        self._ref_text = ref_text or ""

    def _gen_kwargs(self, text, speed) -> dict:
        kw = {"text": str(text or ""), "speed": float(speed)}
        if self._voice is not None:
            kw["voice"] = self._voice
        return kw

    # ---- synthesis ---------------------------------------------------------
    def generate(self, text, speed=1.0):
        t0 = time.time()
        parts = [_extract_samples(c)
                 for c in self._model.generate(**self._gen_kwargs(text, speed))]
        parts = [p for p in parts if p.size]
        full = np.concatenate(parts) if parts else np.zeros(0, dtype=np.float32)
        return full.astype(np.float32), int((time.time() - t0) * 1000)

    def generate_stream(self, text, speed=1.0):
        for chunk in self._model.generate(**self._gen_kwargs(text, speed)):
            samples = _extract_samples(chunk)
            if samples.size:
                yield samples

    # ---- lifecycle ---------------------------------------------------------
    def _clear_ref_file(self) -> None:
        if self._ref_path:
            try:
                os.remove(self._ref_path)
            except OSError:
                pass
            self._ref_path = None

    def unload(self) -> None:
        self._model = None
        self._clear_ref_file()
        self._voice = None
        self._ref_text = ""

    @property
    def is_loaded(self) -> bool:
        return self._model is not None
```

Revisit only if a future change gives MLX rows a distinct quant (see the quant-choice checklist item and the P3 post-plan note).
